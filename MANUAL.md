## ⏰ 행사 1시간 전 — 사전 준비

### 장비 체크
- [ ] 노트북 2대 — 충전 100%, 전원 어댑터 챙김
- [ ] HDMI 케이블 / 디스플레이 어댑터
- [ ] 백업용 휴대폰 핫스팟 가능한지 확인

### 셋업
- [ ] **노트북 A** (큰 스크린 연결용) → 브라우저에서 `/display` 열기
  - 큰 화면에 QR + 응답 현황 잘 표시되는지 확인
  - QR 크기 충분히 큰지 확인 (멀리서도 보여야 함)
- [ ] **노트북 B** (관리자용) → 브라우저에서 `/admin/dashboard` 열기
  - 비밀번호 `0629` 입력 → 로그인
- [ ] 행사장 Wi-Fi에 두 노트북 모두 연결

### 동작 시뮬레이션 (마지막 점검)
- [ ] 본인 휴대폰으로 큰 화면의 QR 스캔 → 접속되는지 확인
- [ ] 본인 이름으로 설문 1회 끝까지 완료 → 결과 카드 확인
- [ ] 관리자 페이지에서 본인 응답 보이는지 확인
- [ ] 본인 응답 **삭제** 버튼 클릭 → 깨끗하게 정리

## 🟢 행사 진행 — 시간별 액션

### 1단계: 입장 + 응답 수집 (참가자 입장 ~ 모두 응답 완료)

- [ ] 큰 화면에 **QR 모드** 띄워놓기 (기본값)
- [ ] 참가자에게 안내: *"앞에 QR 코드 스캔해서 캐릭터 테스트 해보세요"*
- [ ] 관리자 페이지에서 응답 카운트 모니터링 (`X / 38` 형식)
- [ ] 응답이 안 오는 참가자가 있으면 옆에 가서 도와주기

**참고:** 응답 완료 후 새로고침하면 자기 결과 카드가 그대로 보입니다. 같은 사람이 중복 응답할 수 없게 막혀있어요.

### 2단계: 팀 생성 (모두 응답 완료 시점)

- [ ] 관리자 페이지에서 `🎲 라운드 1~4 일괄 생성` 클릭
- [ ] 확인 모달 → "계속" 클릭
- [ ] 약 1~3초 대기 → "완료" 알림 확인
- [ ] (선택) 관리자 페이지에서 `📥 CSV 다운로드` → 중간 백업 보관

### 3단계: 라운드별 진행

각 라운드마다 동일한 패턴:

#### 라운드 1
- [ ] 관리자 페이지 → **모드 변경 → "라운드 1"** 클릭
- [ ] 큰 화면이 자동으로 라운드 1 팀 표시로 전환됨
- [ ] 참가자들에게 본인 팀 찾아가도록 안내
- [ ] 활동 시작 → 약 30~40분

#### 라운드 2
- [ ] 활동 종료 신호 후
- [ ] 관리자 페이지 → **모드 변경 → "라운드 2"** 클릭
- [ ] 큰 화면 자동 전환 → 새 팀으로 이동

#### 라운드 3 / 4
- [ ] 같은 방식 반복

### 4단계: 행사 종료

- [ ] 모드 변경 → "QR 모드"로 되돌리기 (마무리 멘트 동안 깔끔하게)
- [ ] 관리자 페이지 → `📥 CSV 다운로드` → 최종 데이터 보관
- [ ] (선택) 데이터를 행사 후에도 보존하려면 그대로 두기
- [ ] (선택) 데이터 정리하려면 `🗑️ 전체 초기화` 클릭

## 🚨 비상 대응

### 🔴 행사장 Wi-Fi가 끊겼다
- 본인 휴대폰 핫스팟 켜기
- 노트북 2대만 핫스팟에 연결
- 참가자는 본인 LTE/5G 데이터로 접속 가능 (사이트가 매우 가벼움)
- 38명이 동시에 핫스팟 1대에 붙기는 무리

### 🔴 사이트가 안 열린다
- [vercel.com/status](https://www.vercel-status.com/) 확인
- [status.supabase.com](https://status.supabase.com) 확인
- 둘 다 정상인데도 안 되면 → 브라우저 새로고침 / 다른 브라우저 시도
- 그래도 안 되면 → 인쇄해둔 명단으로 무작위 추첨 진행 (최후의 수단)

### 🟡 응답이 너무 적다 (예: 절반만)
- 그래도 `라운드 1~4 일괄 생성` 가능 — 현재 응답 인원으로 자동 균형 잡힘
- 팀당 인원이 줄어들 뿐

### 🟡 참가자가 실수로 잘못 답했다
- 관리자 페이지 → 해당 응답자 옆 **삭제** 버튼 클릭
- 참가자에게 다시 접속해서 응답하라고 안내 (브라우저에서 localStorage 초기화 필요할 수 있음 — 시크릿 모드 권장)

### 🟡 큰 화면이 라운드 전환 안 된다
- 관리자 페이지에서 모드 다시 한 번 클릭
- 큰 화면 브라우저 F5 (새로고침)

### 🟡 팀 배정 결과가 마음에 안 든다
- `🎲 라운드 1~4 일괄 생성` 다시 클릭 (기존 팀이 삭제되고 새로 생성됨)
- 알고리즘에 랜덤 요소가 있어 매번 결과가 다름

## 📥 행사 후 정리

### 데이터 백업
- [ ] 관리자 페이지에서 `📥 CSV 다운로드` → 안전한 곳에 보관
- [ ] CSV 컬럼: 이름, 캐릭터, 4축 점수, 자가인식, 카페 취향, 최애 AI, Q1~Q10 답변
- [ ] 추후 팀 빌딩 참고 자료로 활용 가능

### 데이터 정리 (선택)
- 그대로 보존하려면 → 아무것도 안 해도 됨 (Supabase 무료 티어로 영구 보관)
- 정리하려면 → 관리자 페이지 `🗑️ 전체 초기화`

---

## 💡 기억하면 좋은 팁

1. **모드 전환은 모두 관리자 페이지에서** — 큰 화면 노트북에 직접 입력하지 말기
2. **새로고침 두려워하지 말기** — 모든 데이터는 Supabase에 있어서 페이지 새로고침해도 다 살아있음
3. **다음 라운드 가기 전 활동 시간 충분히 주기** — 참가자들이 새 팀과 친해질 시간 필요
4. **본인이 먼저 시뮬레이션 한 번** — 행사 전에 더미 데이터로 라운드 1~4 미리 돌려보고 화면 익혀두기
